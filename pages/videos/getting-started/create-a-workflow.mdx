---
title: Create a Workflow
---

Learn two ways to build your first workflow:  
one using **Builder Copilot** for speed and one by **manually creating** each step.

## Outcomes
- Build and publish a simple workflow  
- Use connections and variables safely  
- Review run logs and troubleshoot issues  

## Prerequisites
- Blink tenant access  
- At least one connection configured  
- Basic JSON familiarity

  ### Steps
  1. Open **Workflows** and click **New**.  
  2. Choose an **On-demand** trigger for quick testing.  
  3. Click **Builder Copilot** and describe your goal in one sentence.  
  4. Review the suggested steps and refine if needed.  
  5. Add **Connections** to authenticate external calls.  
  6. Create **Variables** with default values as needed.  
  7. **Test** the workflow and check **Run logs**.  
  8. Click **Publish** when ready.

  </Tab>

  ### Steps
  1. Open **Workflows** and click **New**.  
  2. Add a **Trigger** (for example, On-demand).  
  3. Add your first **Action**, such as a transform or HTTP call.  
  4. Attach a **Connection** for authentication.  
  5. Define **Variables** to pass data between steps.  
  6. Add more **Actions** and test each one.  
  7. View **Run logs** to verify success or catch errors.  
  8. Click **Publish** when complete.

  </Tab>
</Tabs>

## Common next steps
- Read from a **Table** to enrich your data  
- Add **If** or **Switch** blocks for conditional logic  
- Convert reusable logic into a **Sub-workflow**

## Troubleshooting
- **401/403** → Check the connection’s permissions  
- **Timeouts** → Run from a **Private runner** if needed  
- **Invalid JSON** → Validate payload structure
